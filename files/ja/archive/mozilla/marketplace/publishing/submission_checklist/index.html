Firefox Marketplace へアプリの登録を始める前に、処理を完了するための全てのリソースを持っているか確認しておくことは有益です。このページには、必要な項目のリストと、関連情報へのリンクがあります。

登録チェックリスト

Firefox Marketplace へアプリ登録を完了するには、下記が必要です:

パッケージ型アプリ用

  • マニフェスト をルートに置き、1つのzipファイル内にパッケージされたアプリ
  • マニフェストで指定された launch_path

ホスト型アプリ用

  • アプリ用のWeb ホスティング。より詳しくは、アプリのホスト用のオプションを見よ。
  • アプリ用のマニフェストファイルのURL。より詳しくは、アプリのマニフェストを見よ。

Firefox Marketplace に登録する前に、アプリのマニフェストをTest App Validation ツールを使って検証できます。

全アプリ用 (必須)

ホスト型アプリとパッケージ型アプリの両方に必要なものは:

  • アプリのマニフェストファイル内の、アプリ名称: マニフェストには最大128文字が入り、これがアプリをアップロードするプロセスの Step 3 にて、Marketplace の名称フィールドに事前入力されます。
  • アプリのマニフェストファイル内の、アプリ説明: マニフェストには最大1024文字が入り、これがアプリをアップロードするプロセスのStep 3 にて、Marketplace の説明フィールドに事前入力され、そこで更に編集できます。(Marketplace 版では文字数についての制限はありません)
  • marketplaceで表示するための 128 x 128px のアイコン; これは唯一の強制的なアイコンサイズです。アプリにアイコンを同梱するための情報は、manifest icons を見て下さい。
  • アプリのスクリーンショット、最小で 320 x 480px のサイズ(より詳細なガイドラインは Marketplace スクリーンショット条件 を見て下さい。なおスクリーンショットは、追加・変更する必要がある場合、ステップ 4 にて変更できます)。
  • アプリのプライバシー条件、あるいはアプリそれを記述したページのURL。より詳細な情報は プライバシーポリシーガイドライン を見て下さい(これは ステップ 3 のアプリのアップロードで入力されます)。
  • アプリのサポート用emailアドレス (ステップ 3 で入力されます)。

有料アプリ、またはアプリ内課金を使うアプリ

  • 未作成の場合、課金事業者、口座番号を含む銀行口座の詳細、SWIFT コード、そして住所の詳細と会社の詳細、例えばVAT 登録や、会社の登録番号と、ビジネス開始日付。
  • アプリがアプリ内課金を行う場合、確実にマニフェストファイル内にオリジンを記入していること。

全アプリ用 (オプション)

ホスト型アプリとパッケージ型アプリいずれも、次のものを持つことが推奨されます:

  • アプリのメインスクリーンを表示する、その他のスクリーンショット。登録ごとに、最大6つまでのスクリーンショットを置けます。理想的には、あなたのアプリが固有のレイアウトを持つフォームファクタごとに持つべきです。例えば:
    • 電話: 推奨サイズ 320 x 480px, またはその倍数
    • タブレット: 推奨サイズ 1024 x 768px や 1280 x 800px
    • デスクトップ: 推奨サイズ 1280 x 800px や 1440 x 900px
  • その他のプラットフォームやコンテキストに最適化するために、その他のサイズのアイコンと、マニフェストファイルでの指定 :
    • 端末のディスプレイ上の 60 x 60px アイコン
    • 様々な他のプラットフォーム、例えばWindows 7 や Android のディスプレイに最適化するための、32 x 32px や 90 x 90px や 120 x 120px や 256 x 256px のアイコン
  • アプリのホームページや、アプリのサポートサイトの URL。これは設定次第(Step 3 のアプリをアップロードする、に入力されます)では、同一だったり、異なっていたりすることがあります。
  • アプリマニフェスト内のその他のオプション項目。名前、説明、アイコン の項目だけが必須項目であり、その他多数のオプションがあります、例えば:
    • アプリが複数言語対応している場合、ロケール 情報を入れたくなるでしょう。
    • ロケールを入れた場合、default_locale 項目は必須です。
    • アプリ内で、特定の限定された端末API や Web アクティビティを使いたい場合、マニフェスト内でそれぞれの アプリパーミッション や アクティビティ を入れておきます。

その他

... こちらも必要でしょう:

  • あなたのアプリが承認後すぐに Marketplace に送られるかを決定します。デフォルトではアプリはすぐに送られます。これを望まない場合は、アプリ登録時にPublish my app in the Firefox Marketplace as soon as it's reviewed(レビュー完了後すぐにFirefox Marketplaceにアプリを発行する)のチェックを外します(Step 2のアプリのアップロードで選択されます)。 
  • Marketplace レビュー条件 (Step 1)を読んで、理解しておきます。
