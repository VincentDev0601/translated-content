Les filtres peuvent être utilisés pour deux usages :

  1. le filtrage, dans lequel l'utilisateur peut visualiser un ensemble unique de données de différentes manières. Par exemple, l'application Calendrier utilise des filtres pour sélectionner l'échelle de temps utilisée de la visualisation de données (qui est de jour, semaine ou mois) ;
  2. la navigation (présentation d'un second ensemble d'onglets lorsque des onglets sont déjà présents dans votre interface utilisateur).

Consultez le Guide de codage pour la façon de mettre en œuvre les filtres dans votre application.

Caractéristiques

  • Les filtres sont présentés comme une séquence horizontale de boutons.
  • Un seul bouton est sélectionné à la fois.
  • La meilleure pratique consiste à placer des filtres dans une barre d'outils, afin qu'ils ne se mélangent pas avec le contenu.
  • Les boutons gauches, droits et centraux peuvent être décorés de façon différente. Cela vous permet par exemple de faire en sorte que les extrémités gauches et droites de votre barre prenenent un «éclat», ou de faire la barre arrondies aux extrémités.
  • Les largeurs de boutons de filtre varient en fonction du nombre de filtres dans un seul ensemble.
  • Vous devez avoir au moins deux et pas plus de cinq filtres dans un ensemble.
  • Une série de filtres peut être marqué avec du texte ou des icônes, mais pas les deux. Parce que les hauteurs de filtres sont relativement faibles par rapport à onglets, le texte est généralement la meilleure solution.
