이 문서는 버전 1.7(Firefox 1.0과 함께 들어간 Gecko 버전) 이래로 Gecko의 XMLHttpRequest 구현에 만들어진 변화를 설명한다. 이 변화는 XUL확장과 XUL응용에만 적용한다. 웹 응용프로그램에 적용하지 않는다.

XMLHttpRequest.send 변화

send method에 nsIInputStream instance를 보내면, 스트림의 시작에 더 이상<tt>Content-Length</tt>와 <tt>Content-Type</tt>을 포함하지 않아야 한다. 대신, <tt>Content-Length</tt> header 는 스트림의 길이로부터 추론되고, <tt>Content-Type</tt> header 는 setRequestHeader method를 불러 수동으로 서술되어야 한다.이 요구는 nsIXMLHttpRequest.idl에서 좀더 전체적으로 설명된다.

XMLHttpRequest.onreadystatechange 변화

이 속성은 nsIOnReadystatechangehandler 대신 nsIOnReadyStateChangeHandler 형식을 가진다.
