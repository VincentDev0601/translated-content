Firefox 3 支援 Animated PNG

  • Animated PNG Specification 的發展
    • 2004 年，Mozilla Corporation 的 Stuart Parmenter 與 Vladimir Vukicevic，依據 Portable Network Graphic (PNG) 而設計了 APNG 延伸規格 (extension)，加入了對動畫的支援。
    • 2006 年，Google Summer of Code 替目前被廣泛使用的 libpng 加入了 APNG 的支援。
    • 2007 年 5 月 23 日，Firefox 3 trunk build 加入 APNG 支援。
  • APNG 與 GIF 的比較
    • 以往網路上的動畫圖片多用 GIF 格式，但因 GIF 每個像素僅支援 8 bit 的色彩深度，所以不適合用於色彩豐富或有漸層的圖片；APNG 支援 24 bit 的色彩深度。
    • GIF 僅支援完全透明、完全不透明；APNG 支援 8 bit 的透明度。
    • GIF 相容性高，目前常見瀏覽器都可以讀取；APNG 設計時即考慮到相容性的問題，所以 APNG 圖片的第一個 frame 是一張靜態的 PNG 圖片，只要瀏覽器支援 PNG，即使無法正確解讀 APNG，仍可顯示這張 APNG 圖片的第一個 frame 內的 PNG 圖片。
  • 講了這麼多 APNG 的好處，要怎麼產生 APNG 圖片呢？APNG Edit、SVG2PNG。
  • 不幸的是，PNG Group 於 2007 年 4 月 20 日正式拒絕將 APNG 納入正式的延伸規格。
